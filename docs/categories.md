# Документация Категорий

## Обзор
Система категорий позволяет организовывать товары в иерархическую структуру (Nested Set). Это обеспечивает эффективный поиск товаров по дереву категорий и генерацию хлебных крошек.

## Структура и Иерархия
Категории поддерживают неограниченную вложенность. Для управления деревом используется пакет `kalnoy/nestedset`.

Пример иерархии:
- **Оборудование** (root)
  - **Газовое оборудование** (child)
    - **Баллоны** (grandchild)

## API Эндпоинты

### CRUD Операции
- `GET /api/v1/categories` - Список всех категорий.
- `GET /api/v1/categories/tree` - Получение дерева категорий (в порядке вложенности).
- `GET /api/v1/categories/{id}` - Детальная информация о категории.
- `POST /api/v1/categories` - Создание новой категории.
  - Поля: `name`, `slug`, `parent_id` (опционально), `description`, `status` (active/inactive).
- `PUT /api/v1/categories/{id}` - Обновление категории.
- `DELETE /api/v1/categories/{id}` - Удаление категории. **Внимание:** удаление родительской категории влечет за собой удаление всех дочерних категорий.

### Навигация и Товары
- `GET /api/v1/categories/{id}/breadcrumbs` - Получение хлебных крошек. Возвращает массив родительских категорий от корня до текущей.
- `GET /api/v1/categories/{id}/products` - Получение товаров категории.
  - Параметр `include_children` (по умолчанию `true`): если включен, возвращает товары текущей категории и всех её подкатегорий.

## Архитектурные решения
- **DDD**: Логика работы с категориями инкапсулирована в доменном слое (`App\Domain\Entities\Category`).
- **CQRS**: Все операции разделены на команды (изменение состояния) и запросы (чтение).
- **Nested Set**: Позволяет получать все товары из поддерева одним SQL-запросом через `whereHas` и `whereIn`.

## Тестирование
Функциональность покрыта тестами в `tests/Feature/CategoryTest.php`, которые проверяют:
- Создание, обновление и удаление категорий.
- Правильность построения дерева.
- Корректность генерации хлебных крошек.
- Фильтрацию товаров по иерархии категорий.
